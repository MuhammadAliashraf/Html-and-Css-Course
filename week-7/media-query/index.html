üì± CSS Media Queries

Media Queries are a powerful feature in CSS that allow you to apply styles based on the device's characteristics such as screen width, resolution, orientation, etc.

üéØ Why Media Queries?

  • To make your website responsive
  • To target specific screen sizes (mobile, tablet, desktop)
  • To enhance user experience across all devices

üîë Basic Syntax

@media media-type and (condition) {
  /* CSS rules */
  }

media-type: screen, print, all (default)
condition: max-width, min-width, orientation, resolution etc.

Example:

@media screen and (max-width: 768px) {
  body {
    background-color: lightblue;
  }
}
Resize your browser to less than 600px width to see background color change!

üìê Common Media Features

  • max-width: triggers if viewport width is less than or equal to specified value
  • min-width: triggers if viewport width is greater than or equal to specified value
  • orientation: portrait or landscape
  • aspect-ratio: ratio of width to height
  • resolution: DPI-based targeting

üß™ Practical Example


/* Base styles */
.box {
  width: 100%;
  padding: 20px;
  background-color: salmon;
  text-align: center;
}

/* Tablet */
@media (min-width: 600px) {
  .box {
    background-color: lightgreen;
  }
}

/* Desktop */
@media (min-width: 992px) {
  .box {
    background-color: lightskyblue;
  }
}

üìù Tips & Best Practices

  • Use em or rem units for better scalability
  • Start with mobile-first using min-width
  • Group media queries logically (e.g., by breakpoints)
  • Test on real devices or use browser dev tools

üìä Common Breakpoints

  • Small devices (phones): max-width: 600px
  • Medium devices (tablets): min-width: 600px and max-width: 991px
  • Large devices (desktops): min-width: 992px

üí° Advanced Example: Orientation

@media (orientation: landscape) {
  body {
    background-color: #ffe082;
  }
}

This changes the background when the device is in landscape mode.

üß† Summary

  • Media queries help create responsive designs
  • They respond to screen width, orientation, resolution, etc.
  • Use min-width for mobile-first strategy
  • Always test responsiveness before shipping!
